# Páginas HTML + Fluxos de Backend

| Página                                         | Fluxo principal                                       | Endpoints relevantes                                                                                                                                                                                                                                | Observações                                                                                                  |
| ---------------------------------------------- | ----------------------------------------------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------ |
| `index.html` (portal do paciente)              | Busca profissionais, filtros e agendamentos           | `GET /api/v1/:clinic_id/professionals`, `GET /api/v1/:clinic_id/appointments?patient_id`, `POST /api/v1/:clinic_id/appointments`                                                                                                                    | Gateway inicial e ponto de partida para login, mantém dependência de dados públicos e agendamentos pessoais. |
| `pages/login.html`                             | Autenticação paciente/admin                           | `POST /api/v1/:clinic_id/auth/login`, `POST /api/v1/:clinic_id/auth/register` (quando aplicável), `POST /api/v1/:clinic_id/auth/logout`, `GET /api/v1/:clinic_id/auth/profile`                                                                      | Login base de todos os fluxos; exige tratamento de erros 401/403 e cookies HttpOnly.                         |
| `pages/register.html`                          | Cadastro de paciente                                  | `POST /api/v1/:clinic_id/auth/register`                                                                                                                                                                                                             | Validar CPF, email, e senha conforme regras de negócio.                                                      |
| `pages/patient-dashboard.html`                 | Dashboard paciente (agendamentos, exames, pagamentos) | `GET /api/v1/:clinic_id/appointments?patient_id`, `GET /api/v1/:clinic_id/exams?patient_id`, `GET /api/v1/:clinic_id/prescriptions?patient_id`, `GET /api/v1/:clinic_id/auth/profile`                                                               | Exibe seções com pagamentos e status.                                                                        |
| `pages/doctor-dashboard.html`                  | Dashboard profissional                                | `GET /api/v1/:clinic_id/appointments?professional_id`, `GET /api/v1/:clinic_id/professionals/:id/availability`, `GET /api/v1/:clinic_id/professionals/:id/commissions`, `POST /api/v1/:clinic_id/professionals/:id/availability`                    | Exige RBAC, visão apenas dos próprios dados.                                                                 |
| `pages/reception-dashboard.html`               | Agenda e processamento (recepção)                     | `GET /api/v1/:clinic_id/appointments`, `POST /api/v1/:clinic_id/appointments`, `DELETE /api/v1/:clinic_id/appointments/:id`, `POST /api/v1/:clinic_id/appointments/:id/reschedule`, `POST /api/v1/:clinic_id/appointments/:id/cancel`               | Controle completo de atendimento, exige filtros por status e permissões.                                     |
| `pages/admin-dashboard.html`                   | Gestão de usuários, profissionais e relatórios        | `GET /api/v1/:clinic_id/users`, `GET /api/v1/:clinic_id/professionals`, `GET /api/v1/:clinic_id/appointments`, `GET /api/v1/:clinic_id/reports/dashboard`, `GET /api/v1/:clinic_id/reports/financial`, `GET /api/v1/:clinic_id/reports/commissions` | Consolida visões administrativas, exibe indicadores.                                                         |
| `pages/doctors.html`                           | Catálogo público de profissionais                     | `GET /api/v1/:clinic_id/professionals`, `GET /api/v1/:clinic_id/professionals/:id/availability`                                                                                                                                                     | Busca por especialidade/nome, usa filtros e exibe slots disponíveis.                                         |
| `pages/agenda.html`                            | Lista e detalhes de horários                          | `GET /api/v1/:clinic_id/appointments`, `GET /api/v1/:clinic_id/professionals/:id/availability`                                                                                                                                                      | Especialmente para recepção e médicos; depende do status e filtros por data.                                 |
| `pages/users.html`                             | Gestão de usuários (admin)                            | `GET /api/v1/:clinic_id/users`, `PUT /api/v1/:clinic_id/users/:id`, `DELETE /api/v1/:clinic_id/users/:id`, `POST /api/v1/:clinic_id/users`                                                                                                          | Listagem com filtros e RBAC.                                                                                 |
| `pages/slots.html`                             | Seleção de horários para agendamento                  | `GET /api/v1/:clinic_id/professionals/:id/availability`, `POST /api/v1/:clinic_id/appointments`                                                                                                                                                     | Exposição dos slots disponíveis (RN-01 a RN-04) e fluxo de criação.                                          |
| `pages/telemedicine.html`                      | Telemedicina                                          | `GET /api/v1/:clinic_id/appointments?type=online`, `POST /api/v1/:clinic_id/appointments`                                                                                                                                                           | Mostra consultas online e links de chamada.                                                                  |
| `pages/checkout.html`                          | Pagamento e confirmação                               | `POST /api/v1/:clinic_id/appointments`, `POST /api/v1/:clinic_id/appointments/:id/cancel`, `POST /api/v1/:clinic_id/appointments/:id/reschedule`                                                                                                    | Integra com regras RN-05 e CloudWalk mock.                                                                   |
| `pages/appointments.html`                      | Fluxos de agendamento (promoções/agenda multi roles)  | `GET /api/v1/:clinic_id/appointments`, `POST /api/v1/:clinic_id/appointments`, `POST /api/v1/:clinic_id/appointments/:id/reschedule`                                                                                                                | Pode servir diferentes roles, portanto precisa de RBAC e filtros.                                            |
| `pages/pep.html` e `pages/prescription.html`   | Prontuário eletrônico e prescrições                   | `GET /api/v1/:clinic_id/prescriptions`, `POST /api/v1/:clinic_id/prescriptions`, `GET /api/v1/:clinic_id/prescriptions/:id`                                                                                                                         | Profissionais atualizam prescrições e veem histórico.                                                        |
| `pages/lab-dashboard.html`, `pages/exams.html` | Gestão de exames                                      | `GET /api/v1/:clinic_id/exams`, `GET /api/v1/:clinic_id/exams/:id`, `POST /api/v1/:clinic_id/exams`                                                                                                                                                 | Lab tech / médico visualizam e processam resultados.                                                         |
| `pages/financial.html`                         | Relatórios financeiros                                | `GET /api/v1/:clinic_id/reports/financial`, `GET /api/v1/:clinic_id/appointments?status=paid`, `GET /api/v1/:clinic_id/transactions` (quando existir)                                                                                               | Dashboard financeiro, exige filtros por data e status.                                                       |
| `pages/password-recovery.html`                 | Recuperação de senha (futuro)                         | `POST /api/v1/:clinic_id/auth/forgot-password`, `POST /api/v1/:clinic_id/auth/reset-password`                                                                                                                                                       | Ainda sem backend MVP, planejar como futura integração.                                                      |
| `pages/onboarding.html`                        | Experiência inicial para novos usuários               | `GET /api/v1/:clinic_id/auth/profile`, `POST /api/v1/:clinic_id/auth/register`, `GET /api/v1/:clinic_id/professionals`                                                                                                                              | Flow orientado a tours e ajuda.                                                                              |

> Este mapeamento será a referência ao implementar cada página, garantindo os endpoints principais necessários.
